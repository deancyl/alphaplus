### 2. 基金筛选页面

**测试URL**: `http://192.168.1.50:60201/fof/fundFilter`

#### 功能亮点

- **26,801只基金数据库**: 数据规模庞大，筛选功能强大
- **5维度穿透筛选**: 支持多条件组合查询
- **实时排序**: 点击列头即可排序
- **分页性能**: ~7ms响应时间 (后端缓存优化)

#### 发现问题

**🔴 严重: API超时问题**

```
错误类型: AxiosError
错误信息: timeout of 30000ms exceeded
影响API: /api/v1/market/indices
出现频率: 持续发生
触发组件: IndexBar.vue, Dashboard.vue
```

**根本原因分析**:
1. `market/indices` API可能依赖外部数据源(AkShare)
2. 外部数据源响应慢或不可用
3. 没有缓存预热或降级机制
4. 30秒超时设置对某些场景过短

**推荐修复方案**:
```python
# 方案1: 增加本地缓存层
@cache(ttl=300)  # 5分钟缓存
def get_market_indices():
    ...

# 方案2: 实现优雅降级
def get_market_indices():
    try:
        return fetch_from_akshare()
    except TimeoutError:
        return get_cached_indices()  # 返回缓存数据

# 方案3: 使用后台任务预加载
@scheduler.scheduled_job('interval', minutes=5)
def preload_indices():
    cache.set('market_indices', fetch_indices())
```

## 🔍 控制台错误分析

### 错误统计

- **总错误数**: 56
- **总警告数**: 14
- **严重程度分布**:
  - 🔴 Critical: 40 (API超时)
  - 🟡 Warning: 14 (Pydantic弃用警告)
  - 🟢 Info: 2 (其他)

### 错误分类

#### 1. API超时错误 (40次)

**错误模式**:
```javascript
[ERROR] API Error: AxiosError: timeout of 30000ms exceeded
    at XMLHttpRequest.handleTimeout
    @ http://192.168.1.50:60201/src/api/index.ts:11

[ERROR] Failed to fetch indices: AxiosError: timeout of 30000ms exceeded
    @ http://192.168.1.50:60201/src/components/IndexBar.vue:15
```

**影响范围**:
- IndexBar.vue (行情条组件)
- Dashboard.vue (首页看板)

**根本原因**:
- `/api/v1/market/indices` 端点响应超时
- 该API依赖AkShare外部数据源
- 可能的网络问题或数据源不可用

#### 2. Pydantic弃用警告 (3次)

**警告信息**:
```
PydanticDeprecatedSince20: Support for class-based `config` is deprecated
```

**影响文件**:
- `backend/core/config.py:17`
- `backend/schemas/pool.py:36`
- `backend/schemas/favorites.py:18`

**推荐修复**:
```python
# 旧写法 (弃用)
class Settings(BaseSettings):
    class Config:
        env_file = ".env"

# 新写法 (Pydantic v2)
from pydantic_settings import BaseSettings, SettingsConfigDict

class Settings(BaseSettings):
    model_config = SettingsConfigDict(env_file=".env")
```
